# 成熟度分级（L1→L2→L3）

> **来源**: cron-scheduler + task-safety 技能的设计模式

## 分级定义

| 级别 | 名称 | 行为 | 自动操作 | 人类门 | 典型时长 |
|:---|:---|:---|:---:|:---:|:---:|
| **L1** | 🟢 仅报告 | 读取+分析+报告，不写外部 | ❌ 无 | ✅ 必须人类决策 | 1-2周 |
| **L2** | 🟡 辅助 | 读取+草稿/准备，改动人批准 | ⚠️ 仅草稿 | ✅ 必须批准 | 2-4周 |
| **L3** | 🔴 自动 | 完整读写执行，信任可无人 | ✅ 自动执行 | ❌ 无（安全名单仍生效） | 通过验证后 |

## 渐进式发布协议

```
第1-2周: L1 — 仅报告，人类审查质量
第3-4周: L2 — 辅助，人类批准每次操作
第5周+:  L3 — 自动，仅当 L1/L2 零失败后
```

## 自动降级规则

L3 任务一旦失败（输出错误、触碰禁止路径），**自动降级到 L1**，人类必须重新认证。

## 与 STATE.md 的配合

```
每次 L3 运行：
  1. 读取 STATE.md
  2. 检查 Idempotency Keys
  3. 执行 → 更新 STATE.md
  4. 如果失败 → 自动降级到 L1
  5. 如果连续成功30天 → 保持 L3
```
